## 1. 資料庫架構總覽

### 1.1 表格概覽

MoreYou Pulse 資料庫由以下8個核心表格組成，用於支撐社群監聽、情緒分析和預警功能：

| 表格名稱 | 用途 | 列數 | 索引數 | 行鎖定(RLS) |
|---------|------|------|--------|----------|
| `pulse_tenants` | 租戶管理 | 9欄 | 1 | ✓ 關閉 |
| `pulse_data_sources` | 數據源配置 | 12欄 | 2 | ✓ 關閉 |
| `pulse_posts` | 社群貼文存儲 | 16欄 | 3 | ✓ 關閉 |
| `pulse_sentiments` | 情緒分析結果 | 12欄 | 2 | ✓ 關閉 |
| `pulse_topics` | 議題追蹤 | 11欄 | 2 | ✓ 關閉 |
| `pulse_alerts` | 預警規則配置 | 12欄 | 2 | ✓ 關閉 |
| `pulse_industry_lexicons` | 產業詞庫 | 8欄 | 1 | ✓ 關閉 |
| `pulse_post_embeddings` | 文本向量嵌入 | 4欄 | 1 | ✓ 關閉 |

### 1.2 關鍵字段定義

#### pulse_posts (社群貼文)
- `id`: UUID 主鍵
- `tenant_id`: 租戶關聯
- `source_type`: 數據源類型（ptt, dcard, google_maps, youtube等）
- `dedup_hash`: 去重雜湊值（防止重複存儲）
- `published_at`: 發佈時間
- `engagement`: JSON 格式的互動數據（讚、評論、分享等）

#### pulse_sentiments (情緒分析)
- `overall`: 整體情緒（positive, negative, neutral, mixed）
- `score`: 情緒分數（-1.0 到 1.0）
- `confidence`: 模型信心度（0.0 到 1.0）
- `aspects`: JSON 格式的面向分析（品質、價格、服務等）
- `purchase_intent`: 購買意圖布爾值
- `crisis_level`: 危機等級（0-10）

#### pulse_alerts (預警規則)
- `rule_type`: 規則類型（keyword, sentiment_drop, competitor, crisis, volume_spike）
- `conditions`: JSON 格式的條件配置
- `threshold`: 觸發閾值
- `last_triggered_at`: 上次觸發時間
- `trigger_count`: 觸發次數累計

#### 正面情緒 (60%) - 主要驅動因素：

- **購買推薦：** 5篇包含購買意圖信號，建議他人投資
- **滿意度表述：** 高度認可產品和服務表現

**代表性內容示例：**
- "Snap-on工具真的超耐用，雖然價格高但耐用度和握感都是一流的"
- "DRG 158真的是經濟實惠的選擇，油耗表現很不錯"
- "專業機車維修廠，使用的都是Snap-on和Blue Point的專業工具，價格合理"

#### 中性情緒 (20%) - 特徵：
- 客觀陳述事實，未帶有明確情感傾向
- 提供教學性內容，如維修知識分享
- 中立比較多個選項

#### 負面情緒 (15%) - 關注區域：
- **價格疑慮：** 部分用戶反映Snap-on和高端工具價格較高
- **品質差異：** 少數提及假貨或不符預期的產品
- **服務投訴：** 個別機車行服務態度和技術水平不達預期

**需要關注的內容：**
- "Bahco工具價格比Snap-on便宜一些，品質也還可以"
- "便宜工具店但品質堪憂，有些產品看起來像是仿冒品"

#### 混合情緒 (5%) - 特點：
- 既有正面認可又有負面疑慮
